针对该 Qt 项目（点单管理系统）的 MVC 架构以及具体的功能需求，UI 设计应遵循“高内聚、低认知负载”的原则。由于是桌面端（Qt Widgets）应用，建议采用 **左侧导航栏 + 右侧主内容区（`QStackedWidget`）** 的经典现代布局。

以下是适配你现有工程结构（特别是 `views/widgets` 目录）的 UI 视觉与交互设计思路：

---

## 一、 整体布局架构 (Overall Layout Architecture)

为了让系统界面（功能七）结构清晰，推荐使用 `QHBoxLayout` 作为底层主布局：

```
+-----------------------------------------------------------------------+
|  侧边导航栏 (QListWidget / Custom Buttons)                              |
|  [ 🛒 点餐大厅 ]                                                        |
|  [ 📜 历史订单 ]                                                        |
|  [ 👥 排队看板 ]                                                        |
+-----------------------------------------------------------------------+
|  右侧主内容区 (QStackedWidget)                                         |
|  页面 1: 点餐大厅 (菜品分类 + 菜单网格 + 购物车与会员)                    |
|  页面 2: 历史订单与评价 (订单列表 + 复制功能 + 评价多级排序)              |
|  页面 3: 实时排队看板 (预约队列 + 取餐队列 + 刷新状态)                    |
+-----------------------------------------------------------------------+

```

## 二、 核心面板交互与视觉设计

### 1. 点餐大厅面板（集成功能一、二、四）

这是系统的核心高频交互界面，建议划分为三个纵向区域：

* **左侧：菜品分类栏 (`QListWidget` 或 `QToolBar`)**
* **视觉**：采用胶囊形状的按钮或高亮条表示当前选中分类（如：热销、主食、饮品、甜点）。


* **中间：菜品展示流 (`QScrollArea` + `QGridLayout`)**
* **自定义组件**：在 `views/widgets` 中实现一个 `DishCard` 类。
* **功能四（推荐点餐）实现**：若该菜品属于 Top 5，在 `DishCard` 的**左上角或右上角叠加一个绝对定位的 Badge 标签**（如红底白字的“🔥 Top 1”或“推荐”），并在边框加上微弱的金黄色阴影以示区别。
* **元素**：图片占位符、菜品名称、价格、简短说明（功能一）、“加入购物车”按钮。


* **右侧：购物车与会员结算栏 (`QVBoxLayout`)**
* **上半部分**：会员登录/信息区。输入会员号后，动态显示会员等级（如：💎 钻石会员）及当前享受的折扣率（功能二）。
* **中间部分**：购物车列表（`QTableWidget` 或自定义列表），显示已选菜品、数量、单价。
* **下半部分**：价格结算区。清晰对比显示：`原价: ¥XX.XX` -> `会员折扣: -¥X.XX` -> `实付金额: ¥XX.XX`，并加粗显示`本次获得积分: +XX`。



### 2. 历史订单与评价面板（集成功能三、五）

采用 **左右分栏（`QSplitter`）** 结构，方便用户在查看历史订单的同时进行评价管理。

* **左侧：历史订单列表**
* **元素**：订单号、下单时间、总金额。
* **功能五（点餐记忆）实现**：在每个订单条目的右侧，放置一个醒目的 **“🔄 再来一单” (Fast Clone)** 图标按钮。点击后，通过 Controller 层的深拷贝逻辑直接跳转回点餐大厅并填满购物车。


* **右侧：菜品评价与排序看板**
* **顶部筛选区**：放置一个 `QComboBox`（下拉选择框）用于切换排序规则，提供两个明确选项：“⭐ 评分从高到低” 和 “⏰ 时间从新到旧”（功能三）。
* **内容展示区**：使用 `QListWidget` 展示评价条目。每个条目包含：用户ID、五星级评分条（可用 `QProgressBar` 改写或用字体图标 `★★★★★`）、留言文本、评价时间。



### 3. 实时排队看板（集成功能六）

此界面需要突出**信息化与动态感**，建议采用大字号和对比明显的看板设计。

* **布局**：对半分栏（`QHBoxLayout`），左侧为“预约排队进程”，右侧为“现场取餐进程”。
* **核心组件**：
* **当前呼号区**：顶部用极大的绿色/橙色字号显示当前正在服务的号码（如 `A023` 或 `TA005`）。
* **等待队列区**：下方用列表展示后续等待的 3-5 个号码。


* **状态与刷新**：
* 右下角放置一个带有加载动画图标的“🔄 刷新看板”按钮（功能六）。
* 可以增加一个小的进度条或文本提示（如：“数据已于 18:04:25 自动刷新”），提升实时感。

## 三、 Qt 控件选型与自定义组件目录指引

结合你的 `views` 目录结构，推荐将复杂的 UI 拆分为独立的子类，以保证代码的可维护性：

| 页面/组件 | 对应类名 (建议在 `widgets` 创建) | 推荐使用的 Qt 核心控件 | 承载的关键功能 |
| --- | --- | --- | --- |
| **主窗口** | `MainWindow` | `QMainWindow` + `QStackedWidget` | 系统整体界面、导航切换 (功能七) |
| **菜品卡片** | `DishCard` | `QWidget` + `QLabel` + `QPushButton` | 菜品信息、Top 5 推荐角标 (功能一、四) |
| **排队看板** | `QueueWidget` | `QWidget` + `QListView` + `QTimer` | 队列双栏实时显示 (功能六) |
| **评价条目** | `ReviewItemWidget` | `QWidget` + `QHBoxLayout` | 星级与留言展示 (功能三) |

## 四、 样式与用户体验（UX）微调建议

1. **颜色语义化**：
* **推荐/热门**：使用暖色调（如 `#FF4D4F` 烈焰红或 `#FFA940` 橘黄）。
* **价格/结算**：使用加粗的大字号暗绿色或深黑色。
* **排队状态**：正在制作/准备中用灰色，请到窗口取餐用高亮绿色闪烁或呼吸灯效果。


2. **空状态（Empty States）处理**：
* 当购物车为空时，右侧结算栏中间显示灰色的“🛒 购物车空空如也~”。
* 当某菜品没有评价时，评价区显示“💬 暂无评价，快来抢沙发吧！”。
* 这能大幅提升系统的完成度和工业感。


3. **防止 QScrollArea 产生横向滚动条**：
* 在动态向菜单网格（`QGridLayout`）添加 `DishCard` 时，务必设置 `scrollArea->setWidgetResizable(true);`，并把网格的列数限制为固定值（如每行固定 3 或 4 个卡片），确保只产生纵向滚动。
